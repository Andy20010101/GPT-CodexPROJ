First public baseline release of `review-then-codex-system`.

This release publishes the repository in its first cleaned, documented, and validated public form. The project is structured as a three-plane monorepo:

- Control Plane: `apps/orchestrator`
- Review Plane: `services/chatgpt-web-bridge`
- Execution Plane: replaceable local executors coordinated by the orchestrator

It also includes the upstream preparation layer, bounded self-improvement tooling, and the operator documentation needed to understand the supported local workflow.

## Highlights

- Introduces a repo-level skill entrypoint through `SKILL.md` and `agents/openai.yaml`
- Publishes the preparation workflow as a first-class documented layer, including CLI docs, templates, examples, and sample packets
- Publishes the bounded self-improvement and operator workflow docs for the supported local mode
- Hardens the orchestrator and bridge runtime paths with stronger review evidence rules, recovery, daemon/runtime controls, and broader test coverage
- Cleans repository boundaries by removing nested project repos from the monorepo and moving retained imports under `references/legacy/`

## Validation

Release baseline validated with:

```bash
npm run lint

```

## Known Limits

- Local-first operating model only
- Real bridge usage still assumes an already logged-in ChatGPT browser session
- The orchestrator is not a distributed scheduler or HA workflow engine
- Root and workspace packages remain `private: true`; this release does not publish npm artifacts
