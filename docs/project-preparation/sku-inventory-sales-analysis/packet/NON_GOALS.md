# Non-Goals

## Explicitly Out of Scope
- Building an internal dashboard or page as a required v1 deliverable.
- Building file-export workflows as a required v1 deliverable, unless a later validated fallback decision explicitly requires export.
- Expanding v1 beyond ERP into WMS, other internal systems, or multi-platform reconciliation.
- Reworking the project into a broad data platform, sync-governance effort, or generalized analytics foundation.
- Defaulting to non-API fallbacks such as database tables or Excel/CSV ingestion in v1.

## Not Now
- Leadership-specific presentation or a separate management-facing product surface.
- Real-time or near-real-time refresh.
- Additional secondary analytics fields, comparison views, and broad presentation polish.
- Multiple external-system interfaces or broad downstream integration work.
- A later boundary decision to introduce export as a fallback if ERP API validation proves key time-dimension metrics impossible to derive.

## Why These Are Deferred
These items are deferred to keep the first version narrowly focused on the core SKU-level API contract and to avoid letting presentation, platform, or fallback complexity dilute the initial business objective.
