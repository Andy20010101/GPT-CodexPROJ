# MVP Scope

## In Scope

- 一个可放入 OpenClaw 工作区的 skill 目录，至少包含 `SKILL.md` 和必要的参考文档。
- 一个围绕单个商品方向运行的端到端研究流：输入研究请求，浏览 1688 结果页和若干供应商/产品页，输出结构化候选清单。
- 一套明确的供应商研究 rubric，说明第一版会看哪些页面可见信号，如何组织输出，以及何时必须承认“不知道”。
- 一个固定的 Markdown 输出 contract，至少包含研究对象、候选供应商列表、代表产品证据、页面可见信号、未知项和原始链接。

## Out Of Scope For MVP

- 大规模批量抓取整个类目、整站采集、持续化监控或定时任务。
- 自动登录、验证码绕过、代理轮换、反反爬对抗、隐藏接口逆向或平台规则规避。
- 自动联系供应商、自动比价下单、自动建采购单或自动推进采购工作流。
- 多平台支持，比如同时覆盖 1688、阿里国际站、淘宝、拼多多等。
- 新的插件、服务端、数据库或复杂 ETL 管道，除非后续验证证明 skill 方案不够。

## Required End-To-End Flow

1. 操作者提供一个清晰的研究 brief，例如目标商品、价位带、MOQ 偏好、地区或工厂特征偏好。
2. skill 使用宿主应用已有的浏览器或网页工具在 1688 上搜索并浏览候选结果。
3. skill 读取若干产品页和供应商页，提取页面可见信号并按统一 rubric 组织观察结果。
4. skill 产出一份以供应商为主排序的候选清单，每个候选项附带 1 到 3 个代表产品作为证据，并记录链接、可见信号、简短判断与未知项说明。

## MVP Boundaries

- 第一版只保证“单主题研究结果可复用”，不保证“抓取规模”或“数据覆盖率”。
- 第一版允许把无法自动确认的信息显式留空或标记为未知，不要求把每个供应商都补全到采购级精度。
- 第一版优先实现“研究助手 skill”，而不是“全自动爬虫系统”。
- 第一版默认只要求 Markdown 报告；如果后续需要 JSON 或 CSV，属于下一阶段增强而不是当前 MVP 阻塞项。
