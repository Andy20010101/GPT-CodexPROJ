# Project Brief

## Project Name

`1688 Supplier Discovery Skill for OpenClaw`

## One-Sentence Objective

`做一个 OpenClaw skill bundle，让操作者能够围绕单个商品方向在 1688 上完成低频、可审计的供应商研究，并以代表产品作为证据输出结构化候选清单。`

## Why This Exists

- 1688 供应商筛选和产品初筛高度重复，人工浏览页面很耗时，而且容易漏掉可见的供应商质量信号。
- 当前缺少一个可复用的研究流程，导致每次都要重新组织关键词、筛选标准、记录方式和输出格式。
- 如果先把“1688 供应商研究方法”沉淀成 skill，后续再扩展到更深的自动化能力会更稳。

## Primary User Or Operator

- 使用 OpenClaw 或 OpenClaw 类应用做货源调研的采购、选品或供应链研究操作者。

## First Usable Outcome

- 操作者给出一个明确的商品方向、筛选条件和偏好后，skill 能在 1688 上完成一轮浏览器优先的研究，并输出一份候选供应商清单。
- 清单中至少包含：供应商名称、代表产品、页面可见的质量信号、价格或 MOQ 参考、原始链接、以及不确定项说明。
- 输出要明确区分“页面可见事实”“启发式判断”“当前未知”，避免把猜测写成确定结论。

## Constraints

- 第一版是 skill，不是新的服务，也不是通用爬虫平台。
- 第一版优先依赖宿主应用已有的浏览器或网页工具，不默认引入新的后端、代理池、队列系统或数据库。
- 第一版只覆盖低频、单主题的供应商研究流，不承诺大规模批量抓取、长周期监控或反反爬能力。
- 1688 页面可见信号、登录要求、验证码、地区限制和平台规则都可能影响可行性，必须在 packet 中显式保留这些风险。
- 第一版默认输出是 Markdown 研究报告；JSON、CSV 或数据库化结果都不是第一版必需物。

## Notes

- 这个项目的核心不是“尽可能多抓数据”，而是“把供应商研究动作收敛成可复用 skill”。
- “优质供应商”在第一版中只能是基于页面可见信号的启发式判断，不是权威认证或最终采购结论。
- 第一版以“找供应商”为主，以“记录代表产品”作为支撑证据，而不是反过来做成产品情报系统。
